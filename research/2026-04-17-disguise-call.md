- What they do: Media server and real-time production platform
- Clients: The Sphere, Broadway shows, major live venues
- Key product: RenderStream - plugin architecture for integrating rendering engines
- Partnership: Work with Notch (similar to ShaderClaw per Gonzo)
- Current approach: Mostly deterministic - pre-programmed sequences, not real-time AI
- Prior mentions: David and Vita brought them up; Gonzo found them in EASL research

## Key Discussion Points

### 1. How James (Disguise) perceived Etherea
- Initially saw Etherea as a rendering engine (like Odyssey)
- Mentioned Odyssey as comparable - sent us a demo link
- TODO: Get the Odyssey demo link from James or Gonzo and archive it here

### 2. Positioning we landed on

> Etherea is a real-time orchestration platform for live models to support live performers.

- We are additive to what Disguise does, not competitive
- They handle deterministic media serving; we handle real-time AI-driven interactivity
- This is a capability gap they haven't filled yet

### 3. EASL positioning
- James (Disguise) called EASL a "competitor" - we reframe:
  - EASL is NOT trying to replace projection mapping software (Disguise already does that)
  - EASL strength: shader compositing - layering shaders on top of each other easily
  - No other tool does this as easily
  - Light projection mapping features still make sense for EASL (Disguise is expensive)
  - Goal for EASL: not mass adoption, but useful for friends, community, school, ourselves
  - Open source vibe-coded project - not trying to compete at The Sphere level
  - Could ask Disguise for full software access in exchange for building tools on their platform

### 4. Odyssey comparison
- Odyssey demo was more about pre-programmed sequences
- Similar to what Scope is doing, but Scope lacks Disguise distribution network
- The distribution network (Sphere, Broadway, etc.) is the moat

### 5. Notch
- Disguise partner for visual effects
- Gonzo describes as similar to ShaderClaw
- Worth understanding their integration model
